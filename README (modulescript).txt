--[[

    Thanks for using GLights!

    This script contains the most important information to get you started.
    You can find further information in Gybasoft Docs.

    1) All of your lights should be in subfolders in the "Lights" folder. If you
        you need more folders or more panels, you can duplicate one of the existing
        light folders. For example, if you want another washes panel, duplicate the
        "Washes" folder.

    2) You should have only one GLights folder in your game. Having more than one
        folder will cause weird behaviour, especially with other Gybasoft tools.

    3) All light models should be named like this: "Fix1", where "1" is the group
        this fixture is in. The number can go up as high as you wish. Cues use the
        groups to figure out the order of lights.
    
    4) To enable something on the panel, left-click on it. To disable something on
        the panel, right-click on it. If you're on mobile or console, you can tap
        the buttons again to disable them.
    
    5) GLights do NOT require StreamingEnabled to be disabled.
        HTTP requests do not have to be enabled for GLights to function, however
        it's recommended to enable them in order to receive update notifications.
    
    6) If your panels turn white, you most likely messed something up while setting
        up GLights, try undoing whatever you did.
    
    7) If GLights work in Studio, but not in the actual game, you most likely have
        virus scripts in your game. That is not the fault of GLights.

]]

return nil
